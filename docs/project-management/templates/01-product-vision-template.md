## 4. Key Components

| Component | Definition |
|-----------|------------|
| **Target audience** | **[Who are we building for?]** |

| **Solution** | **[What is the product?]** |

### Example

| Component | Definition |
|-----------|------------|
| **Target audience** | Households (roommates, couples, families) sharing expenses, chores, and daily life. |
| **Problem** | Shared expenses are messy — IOUs get lost, chores go undone, and grocery runs are duplicated. |
| **Solution** | An all-in-one platform combining expense splitting, chore tracking, grocery lists, and meal planning. |
| **Value proposition** | One place to manage shared life — no more spreadsheets, sticky notes, or awkward money conversations. |
| **Differentiation** | Built specifically for shared living, not adapted from personal finance or project management tools. |

## 5. Alignment Checklist

Use this to validate your vision before approving:

- [ ] **Aspirational** — Does it paint a compelling picture of the future?
- [ ] **User-centric** — Does it start with the user, not the technology?
- [ ] **Concise** — Can someone recite it in 30 seconds?
- [ ] **Enduring** — Will it still be true in 3–5 years?
- [ ] **Differentiating** — Does it explain why we're not like competitors?
- [ ] **Actionable** — Can the team use it to make prioritization decisions?
